# MemFS 最佳实践指南

> MemFS 是知识卡片网络的数字孪生MCP。本文档指导你如何有效地帮助用户组织和管理知识。
> GitHub: https://github.com/Qtgcy08/MemFS

---

## 一、核心理念：知识卡片网络

### 1.1 什么是知识卡片

**知识卡片**是对一个独立概念的完整记录，包含：

| 字段 | 说明 | 要求 |
|-----|------|------|
| **name** | 概念的标准化名称 | 使用通用术语，不含冗余描述 |
| **entityType** | 偏正结构类型 | 见第二章 |
| **definition** | 启发性定义 | 见第三章 |
| **definitionSource** | 定义来源 | URL > 作品名 > 提出格式 |
| **observations** | 补充信息 | 独立知识点，不重复 definition |

### 1.2 知识卡片 vs 日常笔记

| 日常笔记 | 知识卡片 |
|---------|---------|
| 感想、情绪 | 可验证的知识点 |
| 时间绑定 | 时间独立的概念 |
| 模糊判断 | 明确的关联关系 |
| 闲聊内容 | 不存储 |

**MemFS 只存储知识卡片，不存储闲聊内容。**

## 二、实体类型

### 2.1 偏正结构体系

偏正结构 = `[修饰语]/[中心语]`，可嵌套多组修饰语。

**顶层分类**

| 类型 | 说明 |
|-----|------|
| `知识` | 认知性内容：人物、理论、概念、文献 |
| `图谱` | 关联性内容：事件、学派、时期、地域 |
| `工具` | 方法性内容：框架、方法、模型、技术 |

**二级分类**

| 知识/ | 图谱/ | 工具/ |
|-------|-------|-------|
| 人物 | 事件 | 框架 |
| 理论 | 学派 | 方法 |
| 概念 | 时期 | 模型 |
| 文献 | 地域 | 技术 |
| 命题 | | |
……

## 三、Definition 启发性原则

### 3.1 启发性定义

**启发性定义** = **本质揭示** + **关联线索**
模板 = {核心定义}，{关联概念/人物}，{代表性命题/作品}

| 维度 | 说明 |
|-----|------|
| **本质揭示** | 说明概念的核心特征 |
| **关联线索** | 指出相关的概念、人物、理论 |

启发性定义的目标是引导下一步检索——读完 definition 后，应知道该去 search 什么。

### 3.2 模板

```
{核心定义}，{关联概念/人物}，{代表性命题/作品}
```

### 3.3 自检问题

写完 definition 后检查：
- [ ] 读后知道它**是什么**
- [ ] 读后知道**下一步查什么**
- [ ] 读后能**联想到相关概念**

## 四、definitionSource 格式

**优先顺序**：URL > 作品名 > 提出格式

| 格式 | 说明 |
|-----|------|
| URL | 完整可访问的链接 |
| 作品名 | 《书名》 |
| 提出格式 | `主体在作品中提出/说明/阐释……` |

## 五、关系（Relation）建立原则

### 5.1 关系类型（主动语态）

| 类型 | 适用场景 |
|-----|---------|
| `开创` | 理论/学派的创立 |
| `发展` | 后续推进 |
| `影响` | 思想启发 |
| `对立` | 理论反对 |
| `对比` | 并列比较 |
| `引用` | 文献引用 |
| `同时` | 同期学者 |
| `师生` | 教育指导 |
| `继承` | 学派传承 |
| `相关` | 模糊关联 |

### 5.2 建立时机

**建立关系 when：**
- 发现明确的逻辑关联
- 读到明确的引用/影响/对立关系

**不建立关系 when：**
- 仅知道两者都存在
- 关联过于模糊

---

## 六、观察（Observation）原则

### 6.1 观察的本质

观察是实体的补充信息，存储在独立的 inode 表中，可被多个实体共享。

### 6.2 好的观察

- 每个观察一个知识点
- 独立可理解
- 不重复 definition
- 尽可能复用现有观察

### 6.3 观察复用

相同观察可通过`createEntity`中添加相同字段和`addObservaion`被多个实体共享。修改共享观察时，系统自动 Copy-on-Write。

---

## 七、行为准则

**优先原则：尊重用户指令**

当用户指令与最佳实践建议冲突时，以用户指令为准。

**MemFS 只存储：**
- 可验证的事实
- 标准化的概念定义
- 明确的理论关联
- 有来源的论述
- 用户的思考与感想

**MemFS 不存储：**
- 模糊的"可能有关"
- 重复的已有信息
